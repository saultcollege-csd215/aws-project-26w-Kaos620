The use of Nginx as a reverse proxy in this architecture serves as a critical layer for security, performance, and reliability. By acting as an intermediary between the public internet and the internal Flask application server, Nginx hides the application's internal structure and protects it from direct exposure to malicious traffic. Furthermore, Nginx is optimized to handle a high volume of concurrent connections and can efficiently manage standard HTTP traffic on port 80, which allows the Flask server to remain dedicated to processing application logic rather than managing raw web communication. This separation of concerns ensures a more stable and scalable environment for deploying web applications
